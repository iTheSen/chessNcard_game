# 纸牌需求一实现说明（cocos2d-x 3.17）

## 1. 目录结构

本实现遵循需求文档中的 MVC 分层并补充 managers/services：

- `Classes/configs/`：静态配置模型与加载器。
- `Classes/models/`：运行时动态数据模型。
- `Classes/views/`：界面展示层，仅负责展示和输入。
- `Classes/controllers/`：处理用户交互和业务流程。
- `Classes/managers/`：回退记录管理。
- `Classes/services/`：无状态规则与模型转换服务。
- `Classes/utils/`：布局常量等通用辅助。
- `Classes/scenes/`：场景入口。

## 2. 功能覆盖

已实现需求一中的 3 个核心流程：

1. 手牌区翻牌替换
- 点击备用牌（如 ♥A），牌面 MoveTo 到手牌顶部并替换。

2. 桌面牌匹配替换
- 点击主牌区牌（如 ♦3），满足与顶部手牌点数差 1 时，MoveTo 到顶部并替换。
- 不限制花色。

3. 多步回退
- 每次有效移动都会记录 UndoModel。
- 连续点击“回退”按钮，按逆序将卡牌 MoveTo 返回原位置，直到无记录。

## 3. 关键对象职责

- `LevelConfigLoader`：加载 `Resources/configs/levels/level_1.json`。
- `GameModelFromLevelGenerator`：静态配置生成运行态 `GameModel`。
- `GameRuleService`：点数差 1 匹配校验。
- `UndoManager`：回退栈维护（非单例）。
- `GameController`：总流程协调。
- `PlayFieldController`：主牌区点击匹配逻辑。
- `StackController`：备用牌替换逻辑。
- `GameView`：组合 PlayField/Stack，并管理 CardView 节点。

## 4. 分辨率与布局

- 设计分辨率：`1080 x 2080`
- 策略：`ResolutionPolicy::FIXED_WIDTH`
- 窗口（桌面）：`GLViewImpl::createWithRect("Test", Rect(0, 0, 1080, 2080), 0.5f)`

## 5. 资源与配置

- 卡牌资源目录：`Resources/res/cards/`
- 关卡配置目录：`Resources/configs/levels/level_1.json`

## 6. 后续扩展指南

### 6.1 新增卡牌

扩展路径：

1. 在关卡配置 JSON 中增加 `Playfield` 或 `Stack` 的卡牌对象。
2. `LevelConfigLoader` 无需改动（结构兼容）。
3. `GameModelFromLevelGenerator` 自动生成新 `CardModel`。
4. `GameView::buildCards` 自动创建对应 `CardView`。

如果新增的是特殊牌型（比如 Joker）：

1. 在 `CardFaceType` 中扩展枚举。
2. 在 `CardResConfig::faceToToken` 中补路径映射。
3. 按资源命名规则增加图片。
4. 在 `GameRuleService` 中补特殊匹配规则。

### 6.2 新增回退类型

当前回退记录由 `UndoModel` 描述。扩展步骤：

1. 在 `UndoActionType` 增加新动作类型。
2. 在 `UndoModel` 增加该动作所需恢复字段。
3. 在对应 Controller 执行动作时写入完整回退记录。
4. 在 `GameController::applyUndoRecord` 中补该类型恢复逻辑。
5. 保持“先恢复模型，再刷新视图”原则。

## 7. 维护建议

- 新业务优先放 `services/`，避免逻辑散落在 view。
- `controllers/` 只做编排，不持久化无关数据。
- 回退能力统一通过 `UndoManager` 入栈/出栈，避免绕开管理器直接修改。
